Data Analysis

Daniel Keys Moran clearly outlines the challenges facing any project that involves data when he says that, "You can have data without information, but you cannot have information without data."

Cleaning and Organizing Data

In order to get information, you need data. But before you can say that you 'have' data you need to ensure that:

  • You can extract the data from one or more sources.
  • The data that you extract is clean and well behaved. This usually means that the data files are formatted properly.
  • The data is not only clean, but it is also standardized. This is particularly important if data matching is to take place. It is harder to write matching algorithms if things like phone numbers or addresses are not in a consistent, standard format.
  • There is a well designed place to store the data (a database) that allows for efficient access to the data.
  • There are processes in place to efficiently induct the data into the database.
  • (for applications where the database needs to be updated on a regular basis) There must be processes and procedures in place to ensure that new data may be inducted and merged with the existing database.

The cleansing, standardization and induction of data is critical to your enterprise -- yet it is a function that is often underestimated or, as we've seen system architects do, airily dismissed as "implementation details". Completing an application without attention to these steps is roughly equivalent to opening a new building but neglecting to connect all the plumbing correctly. The building may look nice, but the problems that are accumulating in the basement will soon make the building uninhabitable.
